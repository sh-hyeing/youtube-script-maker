# 🎬 YouTube Script Maker

유튜브 영상 자막을 불러와, 내용을 정리하고 영어/한국어 학습용 스크립트로 가공해주는 웹 기반 학습 도구입니다.

긴 영상 자막을 그대로 읽기 어렵다는 문제를 줄이고 사용자가 더 쉽게 복습하고 정리할 수 있도록  
**자막 추출 → 텍스트 정리 → 스크립트 생성 → PDF 저장** 흐름을 한 번에 처리하도록 설계했습니다.

---

## ✨ 핵심 기능

**📺 유튜브 자막 자동 추출**  
유튜브 링크를 입력하면 자막을 불러와 텍스트 형태로 정리합니다. 긴 자막은 후속 처리에 맞게 자동으로 분할하여 안정적으로 다룰 수 있도록 구성했습니다.

**🤖 Gemini 기반 학습 스크립트 생성**  
가져온 자막을 바탕으로 내용을 정리하고 영어/한국어 학습용 스크립트 형태로 재구성합니다. 긴 자막도 청크 단위로 처리해 한 번에 실패하지 않도록 구현했습니다.

**🔁 다중 API 키 회전 처리**  
Gemini API 키를 여러 개 등록할 수 있고 요청 시 키를 순환 사용하도록 구현했습니다. 사용 중인 키와 대기 중인 키를 UI에서 구분해 보여줍니다.

**⏸️ 중단 / 이어하기 지원**  
스크립트 생성 도중 작업을 멈추고 이후 같은 지점부터 다시 이어서 진행할 수 있습니다. 긴 영상 처리 중 사용자가 흐름을 잃지 않도록 설계했습니다.

**⏳ quota exceeded 자동 재시도**  
Gemini 응답에 `Please retry in XXs`가 포함되면 남은 시간을 읽어 자동 카운트다운 후 같은 지점부터 재시도합니다. 현재 상태, 진행 위치, 자동 재시도까지 남은 시간은 STATUS 영역에서 확인할 수 있습니다.

**📄 PDF 다운로드**  
완성된 학습 스크립트를 PDF로 저장할 수 있습니다. 한글 폰트 임베딩을 전제로 한 교재형 레이아웃으로 구성해 복습 자료처럼 보관하거나 출력하기 좋게 만들었습니다.

**📱 모바일 친화적 UI**  
모바일에서는 `VIDEO / API / VIEW / STATUS` 영역을 접고 펼칠 수 있도록 구성했고 Study Script 본문은 내부 박스 스크롤보다 페이지 전체를 자연스럽게 읽는 방식으로 볼 수 있도록 정리했습니다.

## 🧩 프로젝트 구조

초기에는 Next.js API에서 직접 유튜브 자막을 가져오는 구조로 구현했습니다.  
하지만 배포 환경에서는 YouTube 요청이 안정적으로 처리되지 않아 자막 추출이 실패하는 문제가 발생했습니다.

이를 해결하기 위해 자막 추출 책임을 별도 서버로 분리했고 현재는 아래와 같은 구조로 동작합니다.

```text
Browser
  → Next.js /api/transcript
  → Railway yt-dlp transcript server
  → YouTube
```

| 역할                                                                                                                              | 담당             |
| --------------------------------------------------------------------------------------------------------------------------------- | ---------------- |
| 사용자 입력 처리, /api/transcript 프록시, Gemini 스크립트 생성, PDF 다운로드, UI 렌더링                                           | Next.js / Vercel |
| 자막 추출 전용 서버, yt-dlp 기반 자막 추출, cookies 기반 요청 처리, Deno runtime 기반 YouTube JS challenge 대응, .vtt 자막 정규화 | Railway          |
| Gemini API 키 관리, 키 교체 및 회전 상태 확인                                                                                     | Browser          |

---

## 🔐 보안 및 데이터 처리

- Gemini API 키는 사용자의 브라우저에서만 관리하며 별도 서버에 저장하지 않습니다.
- 유튜브 자막 추출은 Railway에 배포한 transcript 서버에서 수행합니다.
- 추출된 자막 텍스트를 기반으로 브라우저에서 학습 스크립트 생성 흐름을 이어갑니다.
- 서버에는 장기 저장소를 두지 않으며 처리 과정에서 사용하는 임시 데이터는 작업 후 정리하는 방향으로 구성했습니다.

## ⚠️ 문제 상황과 해결 과정

**1. Vercel 환경에서 YouTube 자막 추출 실패**

처음에는 Next.js API에서 직접 유튜브 자막을 가져오는 구조로 구현했습니다. 하지만 배포 환경(Vercel)에서는 YouTube 요청이 안정적으로 처리되지 않아 자막 추출이 실패했습니다.

→ 자막 추출 책임을 Railway의 별도 서버로 분리하고 Next.js는 자막을 직접 가져오지 않고 Railway 서버를 호출하는 프록시 구조로 변경했습니다.

**2. yt-dlp 적용 후 요청 차단 및 429 Too Many Requests**

Railway에 yt-dlp 서버를 올린 뒤에도 YouTube 자막 요청이 429 Too Many Requests로 차단되는 문제가 있었습니다.

→ cookies.txt를 적용해 로그인 세션 기반 요청으로 전환하고 여러 언어를 무작정 동시에 시도하지 않고 필요한 언어 위주로 요청하도록 조정했습니다.

**3. YouTube JS challenge 대응 문제**

yt-dlp가 자막 요청 단계에서 Signature solving failed, n challenge solving failed 같은 문제를 일으키며 포맷 확인에 실패하는 상황이 있었습니다.

→ Deno runtime을 연결해 JS challenge를 처리할 수 있도록 구성하고 yt-dlp + deno 조합으로 실제 .vtt 자막 파일 생성까지 확인한 뒤 서버 구조에 반영했습니다.

**4. Docker 빌드 과정에서 Deno 설치 실패**

Docker 이미지 빌드 중 Deno 설치 스크립트가 실패해 Railway 배포가 막히는 문제가 있었습니다.

→ 설치 스크립트 실행 방식 대신 공식 Deno 바이너리를 활용하는 방식으로 변경해 빌드 시점에 더 안정적으로 runtime을 포함할 수 있도록 수정했습니다.

**5. 자막만 필요하지만 포맷 오류로 전체 요청 실패**

영상 다운로드가 목적이 아닌데도 yt-dlp가 포맷 선택 과정에서 함께 실패하며 전체 요청이 중단되는 문제가 있었습니다.

→ 자막 중심 옵션으로 명령을 재구성하고 실제 .vtt 생성 후 텍스트만 정규화해 사용하는 흐름으로 단순화했습니다.

**6. Gemini 무료 쿼터 초과**

긴 자막을 처리할 때 Gemini 무료 플랜의 quota exceeded 오류가 자주 발생했습니다.

→ 다중 API 키 회전 구조를 구현하고 Please retry in XXs 응답을 읽어 자동 카운트다운 후 재시도하도록 처리했습니다. 또한 중단 / 이어하기 기능을 붙여 긴 작업도 다시 처음부터 하지 않도록 개선했습니다.

## 🚀 사용 방법

**1. 프로젝트 설치**

저장소를 클론한 뒤 의존성을 설치합니다.

```bash
npm install
```

**2. 환경 변수 설정**

프로젝트 루트에 `.env.local` 파일을 만들고 Railway 자막 서버 주소를 설정합니다.

```
YTDLP_TRANSCRIPT_SERVER_URL=https://your-railway-domain
```

**3. 개발 서버 실행**

```bash
npm run dev
```

**4. 사용 흐름**

1. 유튜브 링크 입력
2. 자막 불러오기
3. Gemini 학습 스크립트 생성
4. 필요 시 중단 / 이어하기
5. PDF 저장
